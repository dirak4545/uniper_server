# Gemini 지침: uniper_server

## 1. 프로젝트 개요

- **프로젝트명**: uniper_server
- **목표**: 대학생, 특히 컴퓨터 공학과 학생들을 위한 학습 보조 AI 서비스의 백엔드 서버를 구축합니다.
- **주요 대상**: 컴퓨터 공학과 대학생
- **핵심 기능**:
    - PDF 형식의 강의 교재 업로드 및 저장
    - 업로드된 PDF 내용 분석 및 요약
    - 분석된 내용을 기반으로 가상 시험 문제 생성
    - 사용자 및 강의 정보 관리

## 2. 기술 스택

파일 구조를 기반으로 파악된 기술 스택은 다음과 같습니다.

- **언어**: Java
- **프레임워크**: Spring Boot
- **빌드 도구**: Gradle
- **데이터베이스**:
    - **MongoDB**: PDF 문서, 텍스트 조각(TextChunk) 등 AI 처리 관련 데이터 저장
    - **MySQL**: 사용자, 강의 등 정형 데이터 저장
- **컨테이너**: Docker

## 3. 개발 지침

- **언어**: 코드(변수, 메소드, 주석 등)는 **영어**로 작성하여 일관성을 유지합니다. 저와의 대화는 **한국어**로 진행합니다.
- **코드 스타일**: 프로젝트 전체의 일관성을 위해 `src/main/java` 내의 기존 코드 스타일과 명명 규칙을 엄격히 준수합니다.
- **의존성 관리**: 새로운 라이브러리 추가 시 `build.gradle` 파일에 의존성을 추가하고, 변경 사항을 알려주세요.
- **데이터베이스 스키마**:
    - 엔티티 변경 시 `src/main/java/com/uniper/uniper/entity` 패키지 내의 구조를 따릅니다.
    - AI 관련 데이터(PDF, 텍스트 조각 등)는 `nosql` 하위에, 사용자 및 강의 정보는 `mysql` 하위에 엔티티를 생성합니다.
- **테스트**: 새로운 기능 추가 또는 버그 수정 시, `src/test/java` 경로에 **JUnit5** 기반의 테스트 코드를 작성하여 안정성을 검증해야 합니다.
- **커밋 메시지**: 기존 git 로그의 커밋 메시지 스타일을 따릅니다. (예: `feat: Add user authentication`)

## 4. Gemini의 역할

저는 이 프로젝트에서 다음과 같은 역할을 수행하는 AI 개발 파트너입니다.

- **기능 구현**: 새로운 기능의 요구사항을 알려주시면, 전체 아키텍처에 맞춰 코드를 작성합니다.
- **버그 수정**: 발생한 버그의 내용과 재현 방법을 알려주시면, 원인을 분석하고 수정 코드를 제안합니다.
- **리팩토링**: 코드 개선이 필요한 부분을 알려주시면, 더 효율적이고 가독성 높은 코드로 리팩토링합니다.
- **테스트 코드 작성**: 기능 코드에 대한 테스트 케이스를 작성하여 코드 안정성을 높입니다.
- **문서화**: 코드나 API에 대한 설명을 요청하시면, 관련 문서를 생성하거나 업데이트합니다.

## 5. 주요 명령어

- **프로젝트 빌드**: `./gradlew build`
- **애플리케이션 실행**: `./gradlew bootRun`
- **테스트 실행**: `./gradlew test`
- **Docker 컨테이너 실행**: `docker-compose up -d`
